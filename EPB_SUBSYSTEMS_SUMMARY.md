# Electric Parking Brake (EPB) - Subsystem Architecture Summary

## Top-Level ECU Architecture

The Electric Parking Brake ECU is modelled as a AUTOSAR-inspired software architecture with 4 primary subsystems:

| Subsystem | Responsibility | Safety Level |
|---|---|---|
| **EPB_InputProcessingSubsystem** | Driver input (switch, CAN) debouncing and validation | ASIL-B |
| **EPB_ActuatorControlSubsystem** | Motor PWM control, force feedback, position control | ASIL-D |
| **EPB_CommunicationSubsystem** | CAN/J1850/ISO9141 message formatting, diagnostics on CAN | ASIL-B |
| **EPB_DiagnosticSubsystem** | DTC management, UDS/OBD-II services, self-test | ASIL-B |

## EPB_CommunicationSubsystem

### Responsibility
CAN/J1850/ISO9141 message formatting and parsing, signal encoding/decoding, message ID arbitration and priority management, CRC calculation and validation, and in-frame response handling for OBD-II queries.

### Key AUTOSAR/OSEK Requirements
- **COM-001**: CAN 2.0B compliance, 500kbps nominal rate (ISO 11898-2)
- **COM-002**: Brake command latency ≤20ms (ASIL-B)
- **COM-005**: Communication loss detection ≤100ms (ASIL-B)
- **COM-007**: Bus-off recovery ≤1s (ASIL-B)
- **COM-009**: OSEK NM 2.5.3 network management (ASIL-B)
- **COM-010**: Status message transmission at 100ms period (ASIL-B)

### Safety Mechanisms
- CRC error detection with 99.9% coverage
- Automatic bus-off recovery with fallback control
- Message timeout monitoring with stale data flags
- OSEK Network Management for bus participation

## EPB_DiagnosticSubsystem

### Responsibility
Diagnostic Trouble Code (DTC) generation and storage, ISO 14229-1 UDS service handling, OBD-II mode $01-$09 service support, system freeze frame and snapshot data, and on-board monitoring and self-test results.

### 6 Sylang Files

| File | Type | Purpose |
|---|---|---|
| `EPB_Diag.blk.sy` | Block | Diagnostic trouble codes |
| `EPB_Diag.fml.dgm` | Feature Model | DTC hierarchy and UDS service mapping |
| `EPB_Diag.int.sy` | Interface | OBD-II/CAN diagnostic signals |
| `EPB_Diag.req.sy` | Requirements | ISO 14229-1 UDS compliance |
| `EPB_Diag.risk.sy` | FMEA | Memory failures, DTC corruption |
| `EPB_Diag.seq.sy` | Sequence | Diagnostic session state machines |

### Key Diagnostic Requirements
- **DIAG-001**: ISO 14229-1 UDS compliance
- **DIAG-002**: SAE J1979 OBD-II modes $01-$09
- **DIAG-003**: Minimum 20 DTCs with 3 freeze frames
- **DIAG-004**: Permanent DTC retention through ignition cycles (ASIL-B)
- **DIAG-005**: Session management (default, extended, programming)
- **DIAG-007**: Self-test within 500ms of ignition on (ASIL-B)
- **DIAG-008**: Communication loss detection ≤100ms (ASIL-B)

### Key DTCs
| DTC | Description |
|---|---|
| P0500 | Vehicle Speed Sensor Malfunction |
| P1579 | Brake Pedal Position vs. Throttle Position Excessive |
| P0504 | Brake Switch A/B Correlation |
